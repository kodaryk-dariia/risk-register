STRIDE applied by element type, not uniformly

Threat modeling with STRIDE assigns categories by the type of element in the data-flow diagram, not the same six categories everywhere. Only the mobile app is a process — the sensor, actuator and server are external entities, and the data flows between them carry their own subset. That's why the threat count varies by component below.

DFD element typeApplicable STRIDE categoriesComponents in this system
External entity Spoofing · Repudiation Sensor, Actuator, Server

Where the likelihood & harm values came from

These 31 threats use evidence-based estimates rather than guesses — drawn from BLE security research, medical-device security surveys, and protocol vulnerability studies. High-likelihood values (0.7–0.9) are assigned where practical attacks have been demonstrated against comparable BLE medical devices; moderate values (0.4–0.6) where the attack is plausible but needs more specific conditions. Harm is calibrated against patient-safety impact, from technical nuisance (0.05–0.1) to outcomes that could affect treatment decisions (0.8–0.95).
